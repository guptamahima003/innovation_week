  2. PERSONA DEFINITION SIGNALS
================================================================================

  HOW SIGNALS MAP TO PERSONAS (WEIGHTED SCORING RULES)
  =====================================================

  Each persona is identified by a scoring function applied to cluster
  centroids. The function defines which signals matter most and how they
  combine. Higher score = better match for that persona.

  +---------------------+--------------------------------------------------+
  | PERSONA             | SCORING FORMULA (what matters most)              |
  +---------------------+--------------------------------------------------+
  | Tech Enthusiast     | + tech_affinity                                  |
  |                     | - price_sensitivity                              |
  |                     | Key: HIGH tech, LOW price concern                |
  +---------------------+--------------------------------------------------+
  | Value Hunter        | + price_sensitivity                              |
  |                     | + deal_seeking_score                             |
  |                     | + cart_abandonment_rate                           |
  |                     | Key: price + deals + abandons at full price      |
  +---------------------+--------------------------------------------------+
  | Considered          | + research_depth                                 |
  | Researcher          | + avg_session_duration / 30                      |
  |                     | Key: deep research + long sessions               |
  +---------------------+--------------------------------------------------+
  | Loyalty Power User  | + brand_loyalty_score                            |
  |                     | + lifetime_spend / 20000                         |
  |                     | - propensity_to_churn                            |
  |                     | Key: loyal + high LTV + low churn                |
  +---------------------+--------------------------------------------------+
  | Lapsing Customer    | + propensity_to_churn                            |
  |                     | + days_since_last_visit / 100                    |
  |                     | - total_site_visits_90d / 50                     |
  |                     | Key: high churn + inactive + few visits          |
  +---------------------+--------------------------------------------------+
  | Business Buyer      | + avg_order_value / 1000                         |
  |                     | + lifetime_spend / 20000                         |
  |                     | - deal_seeking_score                             |
  |                     | Key: high AOV + high spend + not deal-seeking    |
  +---------------------+--------------------------------------------------+
  | Impulse Buyer       | - research_depth                                 |
  |                     | - avg_session_duration / 30                      |
  |                     | + cart_abandonment_rate                           |
  |                     | Key: NO research + quick sessions                |
  +---------------------+--------------------------------------------------+
  | Home Upgrader       | + avg_order_value / 1000                         |
  |                     | + 0.5 * tech_affinity                            |
  |                     | - deal_seeking_score                             |
  |                     | Key: mid-high AOV + moderate tech + not deals    |
  +---------------------+--------------------------------------------------+
  | Gift Shopper        | + research_depth * 0.5                           |
  |                     | + price_sensitivity * 0.3                        |
  |                     | + brand_loyalty_score * 0.2                      |
  |                     | Key: balanced moderate signals across all dims   |
  +---------------------+--------------------------------------------------+
  | Student Budget      | + price_sensitivity                              |
  |                     | - lifetime_spend / 20000                         |
  |                     | - avg_order_value / 1000                         |
  |                     | Key: HIGH price sensitivity + LOW spend/AOV      |
  +---------------------+--------------------------------------------------+

  SIGNAL PRIORITY TIERS:

  Tier 1 (Primary differentiators - highest weight):
    tech_affinity, price_sensitivity, research_depth, brand_loyalty_score,
    propensity_to_churn

  Tier 2 (Spending pattern signals):
    avg_order_value, lifetime_spend, deal_seeking_score

  Tier 3 (Behavioral engagement signals):
    total_site_visits_90d, avg_session_duration_min, cart_abandonment_rate

  Tier 4 (Recency & history):
    days_since_last_visit, lifetime_order_count, return_rate

  -----------------------------------------------------------------------
  EXAMPLE 1: Lapsing -> Value Hunter (RE-ENGAGEMENT)
  -----------------------------------------------------------------------

    Customer: "Rachel" | Current Persona: LAPSING CUSTOMER
    Profile before: churn=0.75, days_since_visit=60, visits_90d=5

    What happens over 3 weeks:
      Week 1: Rachel receives win-back email with 15% off
              - Clicks email, browses sale section (3 visits)
              - Searches "TV deals under $500"
      Week 2: Returns organically, browses 4 more products
              - Adds open-box Samsung TV to cart, abandons at shipping
              - Comes back next day, searches "free shipping code"
      Week 3: Visits daily checking for price drops
              - Adds 2 items on sale, abandons one, buys the other

    After 20+ events, profile updates:
      days_since_last_visit: 60 -> 0
      total_site_visits_90d: 5 -> 22
      propensity_to_churn: 0.75 -> 0.53 (reduced 30% due to activity)
      cart_abandonment_rate: 0.45 -> 0.60 (high abandon on non-deals)

    New signal pattern: HIGH price_sensitivity + HIGH deal_seeking +
                        HIGH cart_abandonment = VALUE HUNTER pattern

    RESULT: Lapsing Customer --> VALUE HUNTER
    Dashboard shows persona shift in real-time.

  -----------------------------------------------------------------------
  EXAMPLE 2: Value Hunter -> Loyalty Power User (UPGRADE)
  -----------------------------------------------------------------------

    Customer: "Kevin" | Current Persona: VALUE HUNTER
    Profile before: price_sens=0.82, brand_loyalty=0.30, LTV=$3000

    What happens over 2 months:
      Month 1: Kevin signs up for My Best Buy membership
              - Starts buying regularly (3 orders in 30 days)
              - Buys from multiple categories (audio + computing + gaming)
              - Stops browsing deals section as much
      Month 2: Earns Silver tier, now gets member pricing
              - Buys preferred brands (Samsung, Sony) repeatedly
              - Lifetime spend crosses $8,000
              - Cart abandonment drops as he trusts the store

    After accumulated events trigger reassessment:
      brand_loyalty_score: 0.30 -> 0.65 (repeat brand purchases)
      lifetime_spend: $3000 -> $8000
      propensity_to_churn: 0.30 -> 0.08 (very engaged)
      price_sensitivity: 0.82 -> 0.55 (less price-driven now)

    New signal pattern: HIGH loyalty + HIGH LTV + LOW churn
                        = LOYALTY POWER USER pattern

    RESULT: Value Hunter --> LOYALTY POWER USER

  -----------------------------------------------------------------------
  EXAMPLE 3: Tech Enthusiast -> Tech Enthusiast (STABLE - NO CHANGE)
  -----------------------------------------------------------------------

    Customer: "Alex" | Current Persona: TECH ENTHUSIAST
    Profile: tech=0.88, price_sens=0.15, AOV=$450

    What happens over 1 month:
      - Buys AirPods Pro ($249), iPad Pro ($1,099)

    Confidence actually INCREASES from 0.15 -> 0.19

  -----------------------------------------------------------------------
  EXAMPLE 4: Impulse Buyer -> Considered Researcher (BEHAVIOR SHIFT)
  -----------------------------------------------------------------------

    Customer: "Tanya" | Current Persona: IMPULSE BUYER
    Profile: research=0.15, session_dur=4min, cart_abandon=0.30

    What happens: Tanya is buying a $2,000 camera for a new hobby.
      Week 1: Browses 6 cameras, reads detailed specs on each
              - Session duration jumps to 25 minutes
              - Reads 40+ reviews on Canon EOS R6
              - Compares Canon vs Sony vs Nikon in detail
      Week 2: Watches comparison videos (tracked as long dwell)
              - Searches "Canon R6 vs Sony A7 IV detailed comparison"
              - Adds Canon to cart, removes it, adds Sony, removes it
      Week 3: Finally decides on Canon EOS R6, purchases

    After reassessment:
      research_depth: 0.15 -> 0.55 (massive increase in review reading)
      avg_session_duration: 4 -> 18 min (long research sessions)
      cart_abandonment_rate: 0.30 -> 0.50 (deliberation, not impulse)

    New signal pattern: HIGH research + LONG sessions
                        = CONSIDERED RESEARCHER pattern

    RESULT: Impulse Buyer --> CONSIDERED RESEARCHER
    Note: This could be temporary. If Tanya returns to quick purchases
    after the camera buy, she may shift back after another 20 events.

  -----------------------------------------------------------------------
  EXAMPLE 5: Student Budget -> Impulse Buyer (LIFE CHANGE)
  -----------------------------------------------------------------------

    Customer: "Jordan" | Current Persona: STUDENT BUDGET
    Profile: price_sens=0.80, AOV=$120, LTV=$1500

    What happens: Jordan graduates and gets first full-time job.
      Month 1: Spending pattern changes dramatically
              - Buys AirPods Max ($549) without searching for deals
              - Buys PS5 ($499) on first visit to gaming section
              - Sessions are short (3-4 min), decisive
              - No coupon searches, no "student discount" queries
      Month 2: Continues quick purchases
              - Adds items to cart and checks out in <2 minutes
              - Research depth near zero
              - Lifetime spend jumps from $1,500 to $4,200

    After reassessment:
      price_sensitivity: 0.80 -> 0.40 (no longer budget-constrained)
      research_depth: 0.45 -> 0.12 (quick decisions)
      avg_session_duration: 7 -> 3.5 min (fast sessions)
      avg_order_value: $120 -> $380 (bigger purchases)
      lifetime_spend: $1500 -> $4200

    New signal pattern: LOW research + SHORT sessions + moderate AOV
                        = IMPULSE BUYER pattern

    RESULT: Student Budget --> IMPULSE BUYER

  TRANSITION ARCHITECTURE:

  +------------------------------------------------------------------+
  |                    TRANSITION TRIGGER SYSTEM                     |
  +------------------------------------------------------------------+
  |                                                                  |
  |  The system uses EVENT-BASED triggering, NOT time-based.         |
  |                                                                  |
  |  +-----------------------------------------------------------+  |
  |  |                                                           |  |
  |  |   Customer Event Stream                                  |  |
  |  |   (browse, search, add_to_cart, checkout, page_leave...) |  |
  |  |                                                           |  |
  |  +---------------------------+-------------------------------+  |
  |                              |                                  |
  |                              v                                  |
  |                   +--------------------+                        |
  |                   | Per-Customer Event |                        |
  |                   | Counter            |                        |
  |                   | (resets after each |                        |
  |                   |  reassessment)     |                        |
  |                   +----------+---------+                        |
  |                              |                                  |
  |                    count >= 20?                                 |
  |                     /          \                                |
  |                   YES           NO                              |
  |                   |             |                               |
  |                   v             v                               |

  KEY DESIGN DECISIONS:

  Q: Why event-based instead of time-based?
  A: Time-based (e.g., "reassess every 90 days") misses rapid behavioral
     shifts. A lapsing customer who suddenly becomes very active would
     wait 90 days to be reclassified. Event-based catches this within
     their first active session (20 events ~ 5-10 minutes of browsing).

  Q: Why 20 events as the threshold?
  A: 20 events is approximately:
     - 5-10 product browses + some searches + cart actions
     - Roughly 1 engaged shopping session
     - Enough signal to detect a pattern change
     - Not so low that noise causes false transitions

  Q: How is the threshold configured?
  A: REASSIGNMENT_EVENT_THRESHOLD in config.py (default: 20)
     - Lower (10): More responsive, but noisier transitions
     - Higher (50): More stable, but slower to adapt
     - Production recommendation: 30-50 events

  Q: What prevents flip-flopping?
  A: The blended update formula (60% new behavior, 40% historical)
     creates momentum. A single atypical session won't override
     months of established behavior. Multiple consecutive sessions
     of new behavior are needed to shift the persona.
